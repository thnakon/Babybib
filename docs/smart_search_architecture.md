# Smart Search Architecture & Source Handling

ไฟล์นี้อธิบายถึงสถาปัตยกรรมและหลักการทำงานของฟีเจอร์ **Smart Search (ค้นหาอัจฉริยะ)** ว่าระบบแยกประเภทการค้นหาอย่างไร และสลับการทำงานไปหาฐานข้อมูล (Databases) ใดบ้างเพื่อความแม่นยำและรวดเร็วที่สุด

---

## 🔍 กลไกการแยกประเภทคำค้นหา (Input Detection)
เมื่อผู้ใช้กรอกข้อความในฟอร์ม (`generate.php`) คำค้นหาจะถูกส่งไปยัง `api/smart_search.php` ซึ่งจะวิเคราะห์ (Detect) รูปแบบข้อความก่อนเป็นอันดับแรกในฟังก์ชัน `detectInputType($query)`:

1. **ISBN**: เป็นตัวเลขล้วน 10 หรือ 13 หลัก (เช่น `9780132350884`)
2. **DOI**: ขึ้นต้นด้วย `10.` และมี `/` (เช่น `10.14456/jtir.2023.1`)
3. **URL**: ขึ้นต้นด้วย `http://` หรือ `https://`
4. **Keyword (คำค้นทั่วไป)**: ข้อความที่เหลือทั้งหมด ซึ่งจะถูกวิเคราะห์ต่อว่าเป็น **คำภาษาไทย** `isThai()` หรือ **คำภาษาอังกฤษ (Global)** เพื่อเลือกเส้นทางที่เหมาะสม

---

## 🗂️ การแบ่งหน้าที่ของแต่ละฐานข้อมูลตามเส้นทาง (Routing Strategy)

### 1. 📚 ค้นหาด้วย ISBN (หนังสือ)
จุดประสงค์: หารายละเอียดหนังสือจากเลขมาตรฐานสากล 
- **Google Books API**: (ตัวหลัก) ทำงานได้ดีที่สุดกับหนังสือทั่วโลกและหนังสือภาษาไทยส่วนใหญ่ที่มีข้อมูลในระบบ
- **Open Library API**: (ตัวเสริม) ดึงข้อมูลหนังสือคลาสสิก หรือหนังสือภาษาอังกฤษ 

### 2. 🔬 ค้นหาด้วย DOI (งานวิจัย/บทความวารสาร)
จุดประสงค์: หาจากฐานข้อมูลตัวเลข Digital Object Identifier 
ระบบจะทำการเช็คก่อนว่าเป็น **Thai DOI** (มี Prefix ของ TCI เช่น `10.14456`, `10.58837`) หรือไม่?
- **สำหรับ Thai DOI**: ส่งให้ `CrossRef` -> `OpenAlex` (ข้าม Semantic Scholar เพราะงานไทยไม่ค่อยมีในนั้น)
- **สำหรับ Global DOI**: ส่งให้ `CrossRef` -> `Semantic Scholar` -> `OpenAlex` 
*(ระบบจะทำ Deduplication ทันที หากได้ข้อมูลมาซ้ำกันจากหลายแหล่ง โดยจะเอาข้อมูลแหล่งที่เนื้อหาครบกว่ามาเติม (Merge) กัน)*

### 3. 🌐 ค้นหาด้วย URL (เว็บไซต์)
จุดประสงค์: ค้นหาชื่อเว็บไซต์ ชื่อบทความออนไลน์
- ขูดข้อมูลผ่านฟังก์ชัน **`searchWebsite()`** โดยโหลด HTML มาอ่าน `<title>` และ meta tags (เช่น `og:title`, `og:site_name`, `<meta name="author">`) จากเว็บไซต์ปลายทางโดยตรง 

### 4. 🇹🇭 ค้นหาด้วย Keyword - ภาษาไทย
จดประสงค์: ต้องการความรวดเร็วและตรงกลุ่มมากที่สุด ลดภาระการไปดึงฐานข้อมูลเมืองนอกที่ไม่จำเป็น
ระบบจะรันฟังก์ชันต่อไปนี้พร้อมๆ กันและรวมผลลัพธ์:
1. **`searchThaiJO`**: ฐานข้อมูลบทความวารสารไทยที่ใหญ่ที่สุด (ดึงด้วย DOMScraper รวบรวม Title, Author แยกตาม Class)
2. **`searchOpenAlexThai`**: ใช้ API ระดับโลก แต่วาง Filter พิเศษกำหนดเฉพาะภาษา `language:th`
3. **`searchGoogleBooksThai`**: ค้นหาหนังสือไทยโดยเจาะจง `langRestrict=th`
4. **`searchOpenLibraryByKeyword`**: เป็น Fallback เผื่อ Google Books ติด Rate Limit (HTTP 429)
5. **`searchCrossRefKeyword`**: เป็นแหล่งดึงข้อมูลจากมหาลัยไทยต่างๆ ที่ฝากข้อมูลไว้ใน CrossRef 

### 5. 🌍 ค้นหาด้วย Keyword - ระดับโลก (Global)
จุดประสงค์: ต้องการกวาดข้อมูลขนาดใหญ่โดยเน้นเทคโนโลยีและหนังสือภาษาอังกฤษ
1. **`searchOpenLibraryByKeyword`** และ **`searchGoogleBooksByKeyword`**: สำหรับหา "หนังสือ"
2. **`searchSemanticScholarByKeyword`**: ทรงพลังที่สุดสำหรับการหา "Computer Science" และงานวิจัยวิทยาศาสตร์
3. **`searchOpenAlexByKeyword`**: มหาสมุทรข้อมูลครอบจักรวาล
4. **`searchCrossRefKeyword`**: เก็บตกร่วมกัน

## ⚙️ เทคนิคพิเศษในระบบ (System Optimizations)

1. **Title Normalization (`normalizeTitle`)**: การรวมข้อมูลจากหลายๆ ฐานมาโชว์ในหน้าเดียว มักจะมีปัญหางานวิจัย "ซ้ำซ้อน" เช่น ThaiJO กับ OpenAlex ส่งมาเรื่องเดียวกัน ระบบจึงล้างเว้นวรรคและอักขระพิเศษภาษาไทยออกให้หมดก่อนเปรียบเทียบเสมอ เพื่อกันการ Duplicate (`similarTitles`)
2. **Dynamic Confidence Score**: ระบบจะให้คะแนน (Score) ความน่าเชื่อถือของเนื้อหา (เช่น หาก ThaiJO ส่งข้อมูลมาจะให้คะแนนเต็ม 95 หากเป็น CrossRef จะให้ 88) เพื่อใช้จัดเรียงลำดับการแสดงผลให้ผู้ใช้เห็นของที่ "ตรง" และ "มีคุณภาพ" ขึ้นก่อน
3. **Timeout Guards & Multi-cURL**: ระบบรันหลาย API พร้อมกันในรูปแบบ Asynchronous (`curl_multi_exec`) เพื่อไม่ให้หน้าเว็บค้าง โดยมี Hard timeout หาก API ตัวไหนล่ม (เช่น CrossRef มีปัญหา) ระบบจะตัดทิ้งทันที และโชว์เฉพาะผลของ API ที่รอดชีวิต (พร้อมแจ้ง Error กลับใน Array `source_errors`)
