# Smart Search Architecture & Source Handling (v4.0)

ไฟล์นี้อธิบายถึงสถาปัตยกรรมและหลักการทำงานของฟีเจอร์ **Smart Search (ค้นหาอัจฉริยะ)** เวอร์ชันล่าสุด (v4.0) ซึ่งเน้นความเร็วด้วยการดึงข้อมูลแบบขนาน (Parallel Fetching) และความครอบคลุมของฐานข้อมูลไทย-สากล

---

## 🔍 กลไกการแยกประเภทคำค้นหา (Input Detection)
ระบบวิเคราะห์ (Detect) รูปแบบข้อความในฟังก์ชัน `detectInputType($query)` เพื่อเลือกเส้นทางที่เหมาะสม:

1. **ISBN**: ตัวเลข 10 หรือ 13 หลัก (เช่น `9780132350884`)
2. **DOI**: รูปแบบ `10.xxxx/...` (เช่น `10.14456/jtir.2023.1`)
3. **URL**: ขึ้นต้นด้วย `http://` หรือ `https://`
4. **Keyword**: ข้อความทั่วไป (แยกเป็นภาษาไทย/อังกฤษ เพื่อเลือกฐานข้อมูล)

---

## 🗂️ แหล่งข้อมูลและฐานข้อมูลที่ใช้ (Data Sources)

### 1. 📚 ค้นหาด้วย ISBN (Parallel v4.0)
ใช้ระบบ `httpGetMulti` เพื่อดึงข้อมูลจาก 4 แหล่งพร้อมกัน เพื่อความเร็วสูงสุด:
- **Google Books Thai**: ค้นหาในเซิร์ฟเวอร์ไทยโดยเฉพาะ (`langRestrict=th`)
- **Open Library**: ฐานข้อมูลหนังสือสากลที่ครอบคลุมมากที่สุดแห่งหนึ่ง
- **Google Books Global**: ฐานข้อมูลหลักของ Google ทั่วโลก
- **Library of Congress (LOC)**: 💥 **[NEW]** หอสมุดรัฐสภาอเมริกัน ดีมากสำหรับหนังสือวิชาการและหนังสือเก่า
- **Suggestion Fallback**: หากไม่พบ ISBN เลย ระบบจะเปลี่ยนไปค้นด้วย Keyword แบบขนานเพื่อแนะนำหนังสือที่ใกล้เคียง (พร้อมแถบสีเหลืองแจ้งเตือน)

### 2. 🔬 ค้นหาด้วย DOI (งานวิจัย/วารสาร)
- **CrossRef**: แหล่งข้อมูล DOI หลักของโลก
- **OpenAlex**: ฐานข้อมูลวิจัยยุคใหม่ที่รวมผลงานจากทั่วโลก (รวมถึงมหาลัยในไทย)
- **Semantic Scholar**: ฐานข้อมูล AI-powered (เน้นสายวิทยาศาสตร์และเทคโนโลยี)

### 3. 🇹🇭 ค้นหาด้วย Keyword - ภาษาไทย (Thai-Optimized)
- **ThaiJO**: ฐานข้อมูลวารสารไทย (Scraping ข้อมูลชื่อบทความ, ผู้แต่ง, ปี)
- **ThaiLIS (TDC)**: 💥 **[NEW]** ฐานข้อมูลวิทยานิพนธ์และงานวิจัยไทย (Scraping ข้อมูล Title, Author, Year)
- **OpenAlex (Thai Filter)**: ค้นหางานวิจัยไทยในฐานข้อมูลสากลด้วย `filter=language:th`
- **Google Books Thai**: เจาะจงหนังสือภาษาไทย

### 4. 🌐 ค้นหาด้วย Keyword - สากล (Global)
- **Semantic Scholar / OpenAlex / CrossRef**: สำหรับบทความวิชาการ
- **Google Books / Open Library**: สำหรับหนังสือสากล

## ⚙️ เทคนิคพิเศษในระบบ (Technical Features)

1. **⚡ Parallel Fetching (`httpGetMulti`)**:
   - ใช้ `curl_multi_exec` ในการยิง Request หลาย API พร้อมกัน
   - ลดเวลาการรอ (Latency) เช่น การค้น ISBN จาก 4 แหล่ง ใช้เวลาเท่ากับการรอแหล่งที่ช้าที่สุดเพียงแหล่งเดียว (ประมาณ 5-10 วินาที เทียบกับแบบเดิมที่อาจใช้ถึง 20-30 วินาที)

2. **🧠 Intelligent Data Merging (`mergeBookData`)**:
   - เมื่อได้ข้อมูลจากหลายแหล่ง (เช่น Google และ Open Library) ระบบจะนำมารวมกันอย่างชาญฉลาด
   - เลือกใช้ภาพหน้าปก (Thumbnail) จาก Google Books (เพราะชัดกว่า)
   - เลือกใช้ข้อมูล Edition และเลขหน้าจากแหล่งที่มีข้อมูลครบที่สุด
   - รวมแหล่งที่มาเข้าด้วยกัน (เช่น `google_books+openlibrary`)

3. **🛡️ Timeout Guards & Error Reporting**:
   - กำหนด Hard Timeout (8-10 วินาที) หาก API ใดตอบสนองช้า ระบบจะตัดทิ้งทันทีเพื่อให้หน้าเว็บไม่ค้าง
   - แสดง **Source Error Indicator** หาก API บางแหล่งล่ม (เช่น HTTP 429 Rate Limit) เพื่อให้ผู้ใช้ทราบสถานะ

4. **🚀 File-based Caching**:
   - จัดเก็บผลการค้นหาล่าสุดใน `/tmp` เป็นเวลา 5 นาที เพื่อลดการยิง API ซ้ำเมื่อผู้ใช้กดค้นหาคำเดิม

5. **🔍 Title Normalization**:
   - ล้างอักขระพิเศษและเว้นวรรคก่อนเปรียบเทียบชื่อหนังสือ (`similar_text`) เพื่อป้องกันการแสดงผลลัพธ์ที่ซ้ำซ้อนจากหลายแหล่งข้อมูล
